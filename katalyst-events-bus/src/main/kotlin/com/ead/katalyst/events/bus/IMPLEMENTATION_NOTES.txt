KATALYST-EVENTS-BUS MODULE
==========================

This module is EMPTY and ready for implementation.

According to OPTIMAL_EVENTS_ARCHITECTURE_REDESIGN.md, this module should contain:

1. EventBus.kt
   - Interface for publish/subscribe
   - suspend fun publish(event: DomainEvent)
   - fun register(handler: EventHandler<out DomainEvent>)

2. ApplicationEventBus.kt
   - In-memory implementation
   - Async handling with supervisorScope
   - ConcurrentHashMap listener management

3. EventHandlerRegistry.kt
   - Interface for injectable handler registry
   - InMemoryEventHandlerRegistry implementation
   - Methods: register, getAllHandlers, getHandlers(type), clear

4. EventBusInterceptor.kt
   - Extension point interface
   - beforePublish(), afterPublish(), onPublishError()
   - InterceptResult sealed class

5. EventTopology.kt
   - Wires handlers to the bus
   - Used during DI bootstrap

6. HandlerException.kt
   - Exception types for handler failures

See OPTIMAL_EVENTS_ARCHITECTURE_REDESIGN.md Module 2 section for full specifications.
